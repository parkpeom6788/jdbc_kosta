**07-jdbc-bank-project**

- 계좌 관리 프로그램 요구사항
	사용자는 계좌 개설이 가능하다 - insert 
	계좌 개설시 계좌번호,계좌주명,비밀번호,계좌타입,잔액정보가 저장되어야 한다 ->  입력   
	계좌번호는 유일해야하고 시스템에서 자동발급 하도록 한다  -> 시퀀스 
	-- 위에는 db
	계좌타입은 여행,주택,자기개발로 한정한다  -> check 제약조건   db , java
	최초 계좌 개설시에는 1000원 이상 입금해야 한다 -> java -> create 메서드 1000원 미만이면 이셉션 발생 
						-> 잘못된값을 입력하였습니다.
	
	잔액조회는 계좌번호에 대한 계좌가 존재할 때 가능하고 비밀번호가 반드시 일치해야 한다 
							-> 잔액조회할때 계좌번호 입력해서 계좌번호조회메서드 호출 if true이면 조회가능
							--> 계좌가 존재하지않으면 , 비밀번호일치하지않으면 이셉션 -> throw 
	
	입금액,출금액,계좌 이체액은 모두 0원을 초과해야 한다  - 
	
	입금시에는 계좌번호 , 비밀번호가 일치해야 한다  - update
	
	출금시에는 계좌번호 , 비밀번호가 일치해야 하며 잔액부족시에는 출금시키지 않도록 한다. - exception , update, select
	
	계좌 이체시에는 송금자 및 수금자의 계좌가 존재해야 하고
	이체액은 0원을 초과해야 하며 
	
	송금자는 잔액이 부족해서는 안되며 송금자 비밀번호가 일치해야 한다 - 잔액부족 exception 
	송금 및 수금의 모든 작업이 정상적으로 수행되었을때만 이체처리되도록 한다 - java commit , 이셉션시 rollback
	
	가장 높은 잔액을 가진 계좌 정보를 조회 할 수 있다.  - java 조회 , db 에서 select 서브쿼리 
	계좌 타입별 계좌타입명 계좌수, 평균잔액을 조회할 수 있다    - group by , count , avg(price) ,  java 조회 , db 에서 select
